# Dashboard Mode Requirements

## Core Concept

- ダッシュボードモードは、タワー登壇チャレンジの機能を維持しつつ、ビジネスライクな背景画像上でプレイできるモードにする。
- 挑戦中に背景タブを切り替えられることを特徴にする。
- 背景は3種類を使う。
  - 税務・会計ナレッジ風
  - 国税庁の解説ページ風
  - 条文が多い法令・通達ページ風
- 背景タブを切り替えたとき、初期のNotion風前面UIが残って背景を邪魔しないようにする。
- PDFビューア、回答欄、解説確認、次へ、シャッフル、Excel順に戻す、税務PDFサンプル、背景タブは各背景上で機能する。
- PDFビューアと回答欄は、別パネルとして浮かせるのではなく、背景画像内に描かれているPDF枠・回答枠に重なるように配置する。

## Interaction

- PDF部分は回答欄より広めにし、ユーザーがPDFを読みやすい比率を優先する。
- PDF部分と回答欄部分の境目を左右にドラッグして表示バランスを調整できるようにする。
- 幅調整はダッシュモードの3テーマそれぞれで独立して保持する。

## 2026-06-20 Fine Tuning

- 税務・会計タブでは、PDF表示部分と回答欄を背景内の右上寄りの空白に移動する。
- 条文風タブでは、PDF表示部分と回答欄を大幅に横へ広げ、上下の狭さを改善するため上方向へ移動する。
- 上記の調整後も、PDFと回答欄の境目をドラッグして表示バランスを調整できる状態を維持する。

## 2026-06-20 Panel Adjustment

- PDF部分と回答/解説部分の左上に、十字型の移動ハンドルを置く。
- ユーザーは移動ハンドルをドラッグして、各パネルを自由に移動配置できるようにする。
- PDF部分と回答/解説部分の右上に、横方向の拡大縮尺ハンドルを置く。
- ユーザーは拡大縮尺ハンドルをドラッグして、各パネルの幅と高さを調整できるようにする。
- 背景テーマごとに移動・サイズ調整状態を独立して保持する。

## 2026-06-21 Non-Dashboard Explanation Flow

- ダッシュモード以外のPDFゲームモードでは、解説時に「問題文と見比べる」ボタンで問題PDFと解説PDFを切り替えない。
- 問題回答中は解答・解説PDFを表示しない。
- 正誤判定後のみ、PDF表示エリア内で問題PDFの下に解答・解説PDFを表示する。
- 正誤判定直後は、解答・解説PDFの位置へ自動スクロールしてカメラ標準を合わせる。
- 問題PDFは同じスクロールエリアの上部に残し、ユーザーがスクロールで問題文と解説を見比べられるようにする。
- 通常CSVゲームでも、解説は回答中には表示せず、正誤判定後に問題下の解説へ自動スクロールする。

## 2026-06-21 RPGモード 基本戦闘システム刷新案

- 現在の「制限時間内に敵1体を倒す」形式から、縦5マス・横10マスのグリッド上に複数体の敵を配置し、それらを倒す形式へ変更する。
- 複数体攻撃を前提にするため、敵1体あたりのHPは現行より低めに調整する。
- 攻撃対象は初期状態ではランダムな敵にカーソルが合う。ユーザーが敵を選択するとカーソル対象を切り替えられる。
- 攻撃ダイスを振ると、現在カーソルが合っている敵を基準にダメージを与える。
- スキルツリーは火・水・木の3軸から、陽・陰の2軸へ変更する。
- コインダイスを追加し、主に武器購入・武器強化に使うリソースとする。
- 武器システムを追加する。序盤はバット・槍・剣などの単体攻撃武器を用意し、強化で威力や攻撃範囲を伸ばす。
- 中盤以降は鞭・ブーメラン・鎖鎌など、複数体攻撃できる武器を追加する。
- 序盤の複数体攻撃武器は、単体攻撃武器より低威力にして成長余地を作る。
- 同じ武器を再クリックまたは再購入することで、攻撃力や攻撃範囲を強化できるようにする。

### 2026-06-21 追加決定事項

- 勝利条件は盤面上の敵全滅とする。
- 1ステージは3wave制にする。各waveの敵を全滅すると次waveへ進み、3wave全滅でステージクリア。
- 倒した敵は同wave内で復活しない。
- 攻撃計算は「攻撃ダイス + 装備武器の攻撃力」を基礎ダメージにし、倍率ダイスが出ている場合は最後に倍率をかける。
- 例: 攻撃ダイス400 + 剣Lv3攻撃力1000 = 1400、倍率1.5なら最終ダメージ2100。
- 武器は倍率ではなく攻撃力表記にする。例: 剣 600 -> 800 -> 1000 -> 1500 -> 2000。
- 武器は装備画面で装備しなおす方式にする。
- 初期仕様では装備できる武器は1つだけにする。
- 将来的なスキル案として、武器を2つ装備できるスキルを検討する。
- 陽スキルは攻撃力、範囲、連鎖、コイン獲得、武器強化効率を中心にする。
- 陰スキルは妨害、追加ターゲット、弱体化、クリティカル、盤面操作を中心にする。
- SPダイスは1種類に戻す。
- スキルツリー側でSPを陽SP・陰SPに変換できるようにする。
- SP変換率を 1.0倍 -> 1.5倍 -> 2.0倍 に伸ばせるスキルを追加する。
- 実装武器候補は、バット、槍、剣、鞭、ブーメラン、鎖鎌、ハンマー、弓、扇、大剣とする。

### 2026-06-21 武器範囲・コイン表示・演出方針

- バットは単体攻撃武器にする。
- 剣は単体攻撃武器にし、バットより高攻撃力にする。
- ハンマーは対象の横方向の敵にも低倍率ダメージを与える。倍率例は 0.2倍 -> 0.3倍 -> 0.5倍。
- 大剣は対象の両隣の敵にも中倍率から等倍に近いダメージを与える。倍率例は 0.5倍 -> 0.8倍 -> 1.0倍。
- 槍は縦方向に攻撃する武器にする。
- 上記は目安であり、ゲームバランスに合わせて攻撃力・倍率は調整する。
- 敵はプレイヤーへ直接攻撃しない。敵の役割は制限時間内に倒す対象として扱う。
- コインはコインダイスで獲得し、敵撃破時にも追加獲得できるようにする。
- 敵撃破後は「○○円獲得」のような獲得表示を約5秒出して爽快感を出す。
- RPG画面内に「装備」ボタンを置き、武器ショップ兼装備画面を開けるようにする。
- 武器UIは未所持なら購入、所持済みなら装備、装備中なら強化の流れにする。
- 武器の見た目は基本的にアイコンで表現する。適切なアイコンがない場合は既存アイコンの色変換などで代用する。
- アイコン不足が2〜3種類程度に収まる場合のみ、必要に応じてAI生成アイコンを検討する。
- 攻撃演出は武器ごとに差を出す。例: 打撃感、横方向攻撃パーティクル、縦方向突き、範囲攻撃エフェクト、SE差分。

### 2026-06-21 RPGモード 起動停止バグ修正

- Claude編集後、RPG画面が「読み込んでいます...」で停止する不具合を修正する。
- 原因は `buildRpgScreenDOM()` が古い3パネル/スクロールナビ構造に戻り、現在のタブ式UI・装備モーダル・武器ボタンと不整合になっていたこと。
- DOM構築時に現在存在しない/現在仕様と合わない要素へイベント接続し、RPG初期化が途中停止するリスクがあった。
- `buildRpgScreenDOM()` を現行仕様の「問題 / 解説 / ダイス・戦闘」タブ式UI、装備モーダル、スキルモーダル、現在のイベント接続に復旧する。

### RPGモード コアコンセプト

ドラクエ風ターン制バトルで、**問題の正誤・速度・難易度**がバトル結果に直結する。
タワー登壇チャレンジの問題DBをそのまま流用し、簿記・税務の知識でモンスターを倒していく。

### バトルの基本ロジック案

#### コマンド選択（ドラクエ風4択）

```
▶ たたかう   じゅもん
  どうぐ     にげる
```

| コマンド | 動作 |
|---|---|
| **たたかう** | 問題を解く。正答でダメージ、誤答で被ダメージ |
| **じゅもん** | ためた「詠唱ゲージ」消費で強力スペル発動（問題スキップ権） |
| **どうぐ** | 集めたアイテム（回復薬・ヒント等）を使う |
| **にげる** | 問題をスキップ。確率で失敗（敵に先制される） |

#### ダメージ計算の考え方

- **正答** → プレイヤー攻撃：基礎ダメージ × 難易度係数 × 速度ボーナス
  - 30秒以内 → クリティカル（1.5倍）
  - 連続正答（れんぞくせいかい）→ コンボボーナス蓄積
- **誤答** → 敵の反撃：難易度が高いほど大ダメージ
- **難問正答** → 大ダメージ＋「じゅもんゲージ」増加

#### スペル（じゅもん）案

科目・分野ごとにじゅもんを設定するとテーマに合う：

| じゅもん名 | 分野イメージ | 効果 |
|---|---|---|
| メラ系 | 財務会計・簿記基礎 | 中程度ダメージ |
| ギラ系 | 税務・申告 | 全体攻撃（複数小敵）|
| イオナズン | 管理会計・応用 | 大ダメージ |
| ベホマ | 解説読み直し | HP回復（1回/戦闘） |
| ルーラ | 苦手分野タグ | 次の問題を得意分野に変更 |

### 敵キャラ・ステージ設計案

#### タワーの各フロア → バトルステージに対応

- 1〜5階：ザコ敵（スライム系）→ 基礎問題
- 10〜20階：中ボス → 大問形式（複数問連続）
- 50・100階：ラスボス → 総合問題・時間制限あり

#### 敵の種類と問題難易度の対応

| 敵 | 問題難易度 | 特徴 |
|---|---|---|
| スライム | 易 | HP低い。初心者向け |
| ドラキー | 中 | 連続攻撃してくる（連問） |
| キラーマシン | 難 | 誤答時の反撃が痛い |
| ゾーマ | 超難・複合 | 全科目混合問題。フロアボス |

### ゲーム体験上のこだわりポイント

1. **SE・エフェクト**：正答時に剣を振るアニメ＋ダメージ数字がポップアップ
2. **戦闘BGM**：学習中の没入感を上げる（サブスク音源 or 8bit風）
3. **経験値・レベルアップ**：問題正答数に応じてキャラが成長 → ステータス増加
4. **装備品**：連続正答ストリークや高難度クリアで装備ドロップ → 攻撃力・HP底上げ
5. **セーブ機能**：フロアごとの進捗を保存。途中離脱しても続きから再開
6. **パーティ制（将来拡張）**：複数ユーザーで協力プレイ（英語物語方式）

### タワー登壇チャレンジとの統合方針

- 既存の問題DB（xlsx）をそのまま使用。RPGモードでも同じ問題が出る
- 問題の難易度フラグ（既存データ）が敵の強さに直結する
- 通常モード・ダッシュボードモード・RPGモードを選択画面で切り替える
- RPGモードでのスコア（ゴールドや経験値）は通常モードとは独立して管理する

## 2026-06-20 設計制約の確定 → スキルツリー＋インクリメンタル方向

### 確定した制約

- 既存の問題出題フロー（PDF/CSVアップロード→出題）は変えない
- 速度ボーナスは入れない（問題によって解答時間が大きく異なるため）
- 正答ポイントを蓄積してスキルツリーを解放する形にする
- 動作は軽量に保つ

---

### 採用方針：スキルツリー型インクリメンタル

#### 基本コアループ

```
問題を解く（既存フロー）
    ↓
正答 → SPポイント獲得
誤答 → SPなし（or 敵HPが回復するペナルティ）
    ↓
バトル結果を表示（問題単位 or セット単位）
    ↓
貯まったSPをスキルツリーに投資
    ↓
次の問題でスキル効果が反映される
```

出題・解答フロー自体は一切触らず、「前後に挟むだけ」の設計。

### ダイスシステム設計（確定版）

#### セッション構造

- **制限時間**：40分
- **目標**：20問クリア = セッション勝利
- **1問の流れ**：問題を解く → 正答なら**ダイスを振って攻撃** → 次の問題へ

#### ダイスの種類（3種）

| ダイス種 | 効果 | 解放条件 |
|---|---|---|
| 🎲 **SPダイス** | 出た目の分だけSPを獲得 | 最初から所持 |
| ⚔️ **攻撃ダイス** | 出た目の分だけ敵にダメージ | スキルツリーで解放 |
| ✨ **スキルダイス** | 出た目に応じた特殊効果（ヒント・ミス無効・回復など） | スキルツリーで解放 |

- 最初は**SPダイスのみ**。問題を解くたびにSPが溜まりスキルツリーを育てる
- 攻撃ダイスを解放して初めて「敵を倒す」楽しさが生まれる → 序盤の成長実感を演出

#### ダイスプールと個数解放

- 最初はダイス**1個だけ**（SPダイスD4のみ）。弱い、でも始まり
- スキルツリーで個数を解放：1個 → 2個 → 3個 → 最大5個（目安）
- 正答のたびにプール全体を一括ロール
- 例（中盤）：攻撃D6×2 ＋ スキルD6×1 ＋ SPD6×1 → 4個同時に振る

---

#### ダイスのグレード（面数強化）

同じ種類のダイスでも面数をSPで強化できる。

| グレード | 面数 | 特徴 |
|---|---|---|
| Lv1 | D4 | 出目1〜4。序盤のみ |
| Lv2 | D6 | 出目1〜6。スタンダード |
| Lv3 | D8 | 出目1〜8。強化版 |
| Lv4 | D12 | 出目1〜12。上級者向け |
| Lv5 | D20 | 出目1〜20。高リスク高リターン |

スキルツリーの成長が「ダイスが大きくなる」ビジュアルで直感的に伝わる。

---

#### 役（コンボ）システム

役はスキルツリーで順番に解放していく。

| 役名 | 条件 | ボーナス |
|---|---|---|
| ぞろ目 | 同じ出目が揃う（例：6・6・6） | 攻撃ダメージ ×3 |
| 連番 | 連続した目（例：4・5・6） | 攻撃ダメージ ×2 ＋ SP+100 |
| フルハウス | 3つ＋2つの組み合わせ | 攻撃ダメージ ×2.5 |
| ストレート（大） | 4連番以上（例：3・4・5・6） | 攻撃ダメージ ×4 |
| オール最大 | 全ダイスが最大面 | 「ドラゴンロール」×6 ＋ 全呪い解除 |

役は解放しないと発動しない。序盤はぞろ目のみ → 連番 → フルハウス → … の順にアンロック。

---

#### ダイスロック（Yahtzee方式）

- 振った後、気に入った目のダイスを任意個**ロック**できる
- ロックしたダイスはそのまま、残りだけ振りなおせる
- 「攻撃6が出た！これはキープして残りを振ってぞろ目狙い」みたいな判断が面白い
- スキルツリー序盤で解放（ダイス2個目を解放した直後あたり）

---

#### 呪いダイス（誤答ペナルティ）＋解除ダイス

**呪いダイス💀の動作：**
- 誤答すると呪いダイスがプールに追加される
- 次の正答ロール時に一緒に振られ、出た目に応じたネガティブ効果
  - 例：出目1〜3 → 敵HP回復 / 出目4〜6 → 自分の攻撃力ダウン（次1回）
- **自動解除されない**。専用の「解除ダイス🔓」で特定の出目を出すまで居座る

**解除ダイス🔓の動作：**
- スキルダイスの一種（またはスキルツリーで解放）
- 解除ダイスを振って**出目6（指定数）** が出たら呪いダイスを1個除去
- 出目が低いと解除できず、呪いダイスが残ったまま次の問題へ
- 「呪いを引きずったまま次の問題を解くか、解除を優先するか」という判断が生まれる
